Dumping Models and Textures (RomFS) from the Nintendo 3DS

regginator729 3DS 2018-12-292019-07-17 4 Minutes

This guide shall allow the reader to dump the contents of titles (games and others) in the Nintendo 3DS. The contents of a 3DS title include its models, textures, and audio, among others.

Prerequisites

  • A Nintendo 3DS with either GodMode9 or braindump installed
  • A legal copy of the game (either in CIA format, in game cartridge, or installed in the 3DS itself)

This guide does not cover the installation of GodMode9. Plailect‘s guide allows the user to install custom firmware to the 3DS, a prerequisite for GodMode9.

Dumping an installed title/cartridge using GodMode9

Make sure that GodMode9 is installed and can be launched from startup. From here, PixelSergey‘s guide will be useful. The user would probably want to read Extracting RomFS from a 3DS game cartrige (sic) or Extracting RomFS from an installed title. Please note that, because of the limited speed of the 3DS CPU, as well as the SD card, extracting the RomFS will take a very long time.

Note that I did not mention Extracting RomFS from a .cia/.3ds/.nds file, since extracting said files using a 3DS would be unnecessary: a computer may be used, and is much faster. The only downside of using a computer is that the command line will be required. Those unfamiliar with the command line will want to perform PixelSergey‘s guide for GodMode9.

Another note is that, if the title is installed in the EmuNAND instead of your SysNAND, use the directory EMUNAND SD instead of SYSNAND SD. Usually, it’s the other way around: if such a directory exists, the title is most likely installed in one’s EmuNAND.

The entire RomFS of the title is now in /gm9/out/romfs/, and may be read or manipulated by the reader.

Dumping updates

Some updates bring substantial changes to the game, adding new models, logic, and whatnot. The Welcome amiibo update to Animal Crossing: New Leaf is an example to this. To do so, both the base game (title ID usually starting with 00040000) and the update (for Welcome amiibo, the ID starts with 0004000E) must be dumped.

Upon choosing Copy to /gm9/out for the update title, the user may either choose to copy to a differently named folder, overwrite files in the existing /romfs, or (not recommended) skip duplicate files. The user probably wants either the first option (for investigating both the base game and the patch) or the second (if the original files on the base game are not necessary).

Dumping from a CIA or 3DS file

For those with a CIA or a 3DS file in hand, the romfs may easily be extracted using ctrtool. Knowledge on operating the command line is necessary, since the utility is run from the terminal. For those unfamiliar with the command line, use GodMode9, or follow the steps laid below.

ihaveamac has released guides for extracting from a 3DS and a CIA file using the command line. For those who do not wish to use a command line, refer to PixelSergey‘s guide linked above. The following steps must be performed:

  1. Read the instructions before performing them, to have an idea on what to do. Know that errors may pop up along the way: for example, the 3DS/CIA file may be corrupted or unable to be extracted, or the system is a 32-bit one and ctrtool cannot be run. I cannot help the reader here.
  2. Download the latest release of ctrtool. Download the makerom_....zip file and extract it. Choose the proper ctrtool.exe or ctrtool file depending on the operating system (Windows, Mac, or Linux).
    • The executables are built for 64-bit OS’s. For those using a 32-bit one, the executable must be built from source. More modern systems are 64-bit, so the reader need not worry.
    • Guides on determining whether the system is 64- or 32-bit are available in the Internet.
  3. Make sure that the ctrtool file and the 3DS/CIA file (renamed to game.3ds or game.cia) are in the same folder. Open the command prompt or terminal, and navigate to the folder in question.
    • The more technical audience will understand that the files need not be in the same folder, nor do they need be renamed; nonetheless, it is assumed that the reader is a beginner.
  4. In the command line, run one of the following commands, depending on the reader’s use case:
    • 3DS, Windows: ctrtool.exe --romfsdir=romfs game.3ds
    • 3DS, Mac/Linux: ./ctrtool --romfsdir=romfs game.3ds (replace ctrtool.exe with ./ctrtool)
    • CIA, Windows: ctrtool.exe --contents=contents game.cia and then ctrtool.exe --romfsdir=romfs contents.0000.00000000
    • CIA, Mac/Linux: replace ctrtool.exe above with ./ctrtool

The full RomFS of the title is in the romfs folder with ctrtool, and may be read or manipulated by the reader.

Dumping an installed title/cartridge using braindump and ctrtool

For those without boot9strap installed, but with access to Homebrew (for example, through an Internet browser exploit), it is possible to dump the contents of a cartridge or an installed title. ihaveamac also published this guide for this use case.

Note that this use case is not recommended. A 3DS with custom firmware is recommended (see Plailect‘s guide above).

  1. Install braindump in the /3ds/ folder of the SD card, and launch it.
  2. Choose the title to be dumped to a CXI file. The file will be saved in the root of the SD card.
    1. As written above, the reader may also want to dump the update.
  3. Download the latest release of ctrtool. On a folder, copy the appropriate ctrtool.exe or ctrtool, and the CXI file dumped earlier. Rename the CXI file to game.cxi.
  4. Perform one of the following commands in the terminal, depending on the reader’s OS:
    • Windows: ctrtool.exe --romfsdir=romfs game.cxi
    • Mac/Linux: ./ctrtool --romfsdir=romfs game.cxi

The full RomFS of the title is in the romfs folder with ctrtool, and may be read or manipulated by the reader.
